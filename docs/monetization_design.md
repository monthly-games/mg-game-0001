## 게임 개요
- **장르**: Tower Defense, Strategy
- **타깃**: 캐주얼~미드코어, 전략 게임 선호 유저
- **수익화 우선순위**: **리워드 광고 (50%) > 스킨/장식 (20%) > 편의 기능 (15%) > 기타 (15%)**
- **핵심 철학**: **P2W 최소화**, 광고와 편의/장식 중심의 건전한 수익 모델

## 1. 광고 전략 (50% 수익 기여)

### 1.1 보상형 광고 배치 (Rewarded Ads Only)

타워 디펜스의 핵심 수익원. **강제 광고 없음**, 모든 광고는 유저의 선택.

| Placement ID | 상황 | 보상 | 일일 제한 | 쿨다운 |
|--------------|------|------|----------|--------|
| `td_wave_double` | 웨이브 완료 후 선택 | 골드 획득량 2배 | 10회 | 30초 |
| `td_gold_boost` | 메인 UI에서 선택 | 30분간 골드 +50% | 3회 | 1시간 |
| `td_life_revive` | Life가 5 이하일 때 | Life +5 회복 | 5회 | 1분 |
| `td_tower_upgrade` | 타워 선택 시 | 1개 타워 즉시 업그레이드 | 3회 | 2분 |
| `td_wave_skip` | 웨이브 시작 전 | 해당 웨이브 자동 완료 | 3회 | 5분 |
| `td_daily_bonus` | 메인 메뉴에서 선택 | 500 골드 지급 | 1회 | 24시간 |
| `td_chest_bonus` | 챕터 완료 보상 | 보상 1.5배 | 3회 | 1시간 |

**총 광고 일일 캡: 25회**

### 1.2 광고 설계 원칙

```yaml
Ad_Design_Principles:
  # 절대 규칙
  forced_ads: false                  # 강제 광고 없음
  interstitial_ads: false            # 전면 광고 없음
  rewarded_only: true                # 리워드 광고만 허용

  # 유저 경험 보호
  first_session_ad_block: 3min       # 첫 세션 3분간 광고 차단
  per_placement_cooldown: 30sec      # 플레이스먼트별 쿨다운 30초
  daily_cap: 25                      # 일일 총 광고 시청 제한

  # 광고 제거 구매자
  no_ads_purchase_behavior: "auto_grant_rewards"  # 광고 없이 보상 자동 지급
```

## 4. 재화 시스템 및 밸런스

### 4.1 재화 종류

| 재화 | 타입 | 용도 | 획득 경로 |
|------|------|------|----------|
| **골드** | Soft | 타워 구매/업그레이드 | 웨이브 클리어, 광고, IAP |
| **젬** | Hard | 스킨 구매, 편의 기능 | 첫 클리어 보상, IAP |

**Simple is Best**: 재화를 2종류로 제한하여 유저 혼란 방지

### 4.2 일일 무료 획득량 (무광고 기준)

## 12. 부록: 수익화 FAQ

**Q1: 광고 제거를 구매한 유저는 어떻게 되나요?**
A: 광고 없이 보상만 자동 지급됩니다. 유저 경험이 개선되며, 광고 수익 손실보다 유저 만족도가 중요합니다.

**Q2: 스페셜 타워가 P2W가 되지 않을까요?**
A: 모든 스페셜 타워는 기본 타워 대비 ±20% DPS 이내로 제한하며, 무료로도 모든 웨이브 클리어 가능하도록 밸런스를 설계했습니다.

**Q3: 골드 밸런스는 어떻게 관리하나요?**
A: 주간 골드 획득/소비 비율을 모니터링하며, 인플레이션율 ±10% 이내 유지를 목표로 합니다.

**Q4: 스킨 수요가 부족하면 어떻게 하나요?**
A: 월 1회 신규 스킨 추가, 한정 스킨으로 희소성 확보, 리더보드/이벤트 보상으로 스킨 가치 향상을 통해 수요를 창출합니다.
